物理信道

允许特定信息或数据通过的路径

信道是无需具有可见的、有形状的物理边界，是由一些物理或逻辑属性定义的

参考信号

  • 在无线通信中（尤其是长距离无线通信），所有信号都将经历噪声、衰减环境，并且变形甚至中断
  • 为了从失真变形的信号中解析出正确的信号，发射器发送一个经过定义的信号，并且接收器通过无线环境解析该信号。这种定义的信号叫做参考信号

同步

Sync = Synchronization = Setting the same clock on Sender and Receiver

不同的无线技术使用不同的，特定的信号来实现同步过程

在任何数字通信系统中（甚至是简单的有线通信），都有一种特殊的同步机制。在简单的短距离有线通信系统中，设备通过特定的脉冲信号边沿来同步。在一个有点复杂的有线系统中，它们使用一个通常称为“preamble”的位模式序列进行同步。

在无线通信环境中，影响时间的因素更加多样化。因此我们需要高效和复杂的机制来确保同步。通常在蜂窝系统中，我们使用一种或多种类型的特殊信号进行下行同步，上行同步使用RACH。

小区选择

一种检测和选择最佳小区来注册的机制

  • 测量设备周围小区的信号强度，并且在条件允许的情况下解码SIB
  • 选择最合适的小区进行注册

上述小区选择机制总结如下：

  1. 设备开机上电，或是关闭飞行模式
  2. UE扫描周围所有小区的强度
  3. 如果它检测到功率大于某一电平(该电平在UE radio stack软件中指定，该值将根据UE制造商和型号而变化)，它将解码MIB(主信息块)和带有小区选择参数的某个SIB(系统信息块)。哪个SIB携带这种信息与每种无线电技术不同
  4. 在解码MIB/SIBs之后就可以获得小区选择参数，根据小区选择参数和3gpp协议进行小区选择
  5. 根据以上步骤进行最佳小区选择并注册

小区重选

一种在空闲模式下检测并选择最佳小区的机制

  • 测量UE周围邻小区的信号强度来检测最强小区
  • 选择最强小区进行注册

小区重选和小区选择非常相似，小区重选发生在UE完成小区选择并进入空闲模式之后

上述小区重选机制总结如下：

  1. UE处于空闲模式（有服务状态）
  2. UE扫描周围所有小区的电平
  3. 如果它检测到功率大于某一电平(该电平在UE无线电堆栈软件中指定，该值将根据UE制造商和模型的不同而变化)，它将解码带有小区重选参数的某个SIB(系统信息块）
  4. 对SIBs进行解码，得到小区重选参数后，根据小区选择参数和3gpp协议进行小区重选
  5. 根据以上步骤进行最佳小区重选并注册

功控

开环功控、闭环功控

在蜂窝通信中，根据基站与UE之间的距离以及其他信道条件，基站接收到的功率变化很大。为了保持基站接收到的功率稳定(至少不急剧变化)，蜂窝系统采用非常复杂的算法来控制/调整UE的传输功率。这个算法机制叫做功控Power Control。在大多数蜂窝通信(至少3G、4G、5G)中，主要有两种功率控制机制，即开环功率控制和闭环功率控制。

测量

网络要求UE进行粗略的测量，以达到以下目的：

  • 评估当前链接的信号质量，如RSRP、RSRQ、CQI、RI、PMI…
  • 在建立载波聚合之前，检查目标小区是否存在（可分离），质量是否足够好
  • 在执行切换前，检查目标小区是否存在（可分离），质量是否足够好

在任何情况下，测量命令(从网络到UE的方向)是通过一个特定的RRC消息(3G中的测量控制，4G中的RRC连接重配置)完成的，但是测量报告是根据情况由几个不同的机制完成的

  • 一些测量（CQI、RI、PMI…）是通过物理信道上报的（PUCCH或PUSCH）
  • 一些测量（RSRP、RSRQ、Event）是通过RRC消息上报的

小区测量是蜂窝系统中移动性和信道质量评估，以及自适应调度（如，基于CQI的MCS变化）的非常重要的步骤

切换

  1. A UE is in connection with a cell (let’s call this ‘Cell A’)
  2. Now a situation that requires handover happened
  3. Network send “signal quality measurement” command to UE for the target cell(‘Cell B’) to which it will handover to.
  4. UE performance the measurement and report the “measurement result” to the network via the current cell (Cell A).
  5. Netwok evaluate the measuement result reported by UE
  6. If the evaluation result turns out to be good for handover, Network send ‘Change Cell’ command to UE.
  7. UE perform the cell change process.
  8. If cell change process is completed properly, UE send ‘cell change completion’ message to the network via the target cell (Cell B).

I used very generic term e.g, “signal quality measurement command” , “measurrement result” , “change cell command” , “cell change completion message” etc. These generic commands can be translated to a specific jargon for each technology. For example, if i translate these for UMTS, they would be as follows:

“signal quality measurement command” —> Measurement Control

“measurement result” —> Measurement Report

“change cell command” —> Physical Channel Reconfiguration or ActiveSetUpdate

“cell change completion message” —> Physical Channel Reconfiguration Complete or ActiveSetUpdateComplet

If you translate them into LTE jargon, they will be as follows:

“signal quality measuerment command” —> RRC Connection Reconfiguation

“Measurement result” —> Measurement Report

“change cell command” —> RRC Connection Reconfiguration

“cell change completion” —> RRC Connection Reconfiguration Complete

You may noticed that LTE is using the same message called “RRC Connection Reconfiguration” both for “signal quality measurement command” and “change cell command” . How UE can tell whether it means “signal quality measurement command” or “change cell command” ?
